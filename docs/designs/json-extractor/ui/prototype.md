# JSON Extractor - UI Prototype

## 实现顺序

```
CLI (Rust) → WASM → Web UI → Desktop → Tests
```

## 平台差异说明

| 功能 | Web | Desktop |

| 路径语法帮助 | ✅ (Hover) | ✅ (Hover) |
| 输出格式化 | ✅ | ✅ |
| 复制/下载 | ✅ | ✅ |
| 历史记录 | ❌ | ✅ |
| 收藏路径 | ❌ | ✅ |
| 配置持久化 | ❌ | ✅ |
| 系统托盘 | ❌ | ✅ |

## 核心变更说明

### 1. 输入区域顶部按钮栏

```
[📥 URL 导入] [🗑 清除输入] [📄 格式化 JSON]
```

| 按钮 | 功能 |
|------|------|
| 📥 URL 导入 | 弹出 Modal 输入 URL，点击确认后自动获取并填充 JSON |
| 🗑 清除输入 | 清空 JSON 输入区域内容 |
| 📄 格式化 JSON | 将输入的 JSON 美化/压缩 |

### 2. JSONPath 输入行（右侧整合）

```
JSONPath: [______________________________] [?] [🔍 提取]
         ↑ 输入框                      ↑ 语法帮助  ↑ 提取按钮
```

- 将 "JSONPath 表达式输入框" + "查看可用路径" + "提取按钮" 合并到一行
- 放在输入区域下方，最大化输入空间
- 最右侧放置提取按钮

## 交互设计细节

### 1. URL 导入 Modal

```
┌─────────────────────────────────────┐
│  📥 从 URL 导入 JSON                      │
│  ───────────────────────────────────│
│                                       │
│  URL: [____________________________]  │
│                                       │
│  [取消]              [确认导入]          │
│                                       │
└─────────────────────────────────────┘
```

- 点击 "📥 URL 导入" 按钮 → 弹出 Modal
- 输入 URL → 点击 "确认导入"
- 自动获取 URL 内容，填充到 JSON 输入区域
- 加载中显示 spinner
- 支持自动检测 JSON 或让用户确认

### 2. 清除输入

- 点击 "🗑 清除输入" → 清空 JSON 输入区域
- 同时清空输出区域
- 清空 JSONPath 输入框

### 3. 格式化 JSON

- 点击 "📄 格式化 JSON" → 格式化当前输入
- 有效 JSON → 美化/缩进
- 无效 JSON → 无操作 + 提示 toast
- 支持切换：格式化 / 压缩

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| Ctrl+Enter | 执行提取 |
| Ctrl+L | 清空输入 |
| Ctrl+Shift+F | 格式化 JSON |
| Ctrl+U | 打开 URL 导入 Modal |
| Ctrl+C (output focused) | 复制结果 |
| Ctrl+D | 下载结果 |

## 组件清单

- `UrlImportModal` - URL 导入 Modal
- `JsonInput` - JSON 输入区（带验证）
- `InputToolbar` - 输入区域顶部工具栏（URL/清空/格式化）
- `PathInput` - 路径输入（带帮助按钮）
- `ExtractButton` - 提取按钮
- `OutputPanel` - 结果显示区（带格式化）
- `FormatSelector` - 格式选择器
- `ActionButtons` - 操作按钮组（复制/下载）
- `HelpButton` - 帮助按钮（Hover 显示语法帮助）
